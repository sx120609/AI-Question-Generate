## 根因

旧生产链主要依赖提示词中的“换行业、换口吻、去 AI 味”和题面关键词 lint，缺少四项系统约束：

1. 没有跨 run 的结构使用配额；
2. 没有对五个叙述字段生成任务图指纹；
3. 没有题面长度和有效信息覆盖硬门槛；
4. lint 通过后仍可改稿并直接提交，没有不可绕过的提交凭证。

## 已实施的修复

1. run 创建时自动预留结构护照，轮换长度、入口、信息顺序、决策、证据、产物和流程拓扑。
2. 对同批与历史记录做多维结构比较，行业名词替换和步骤动词同义改写不会规避检测。
3. 题面硬下限为 480 个可见字符，正常计划从 520 字起；核心场景和信息覆盖不足单独失败。
4. 只有结构门禁 `PASS` 才签发哈希回执；正式写入 B/G/L/N/O 前必须验票，并在全局锁内重查后登记为 `reserved`，写入成功再升级为 `submitted`。门禁后改稿会使回执失效，并发 run 也不能拿同一旧快照同时提交。
5. 22 条既有记录已作为 `legacy` 历史基线登记，后续沈礼或裴硬的新题都会与它们比较。

## 迁移建议

既有 22 条不应自动批量拉长，因为那会在没有重新核对附件和事实的情况下制造内容。建议按每批 3–5 条迁移：先确定需要保留的业务事实与附件，再从任务图层重写五个叙述字段，经过结构门禁、附件复核和三轮 QA 后覆盖原 UID 对应行。
