## 当时的第一道质量门禁

1. 明确真实工作或决策场景，包含具体对象、当前约束和一个主要目标。
2. 通常需要 4 至 8 个关键步骤，包含检索、理解、比较、核验、证据边界和初步判断，不能退化为单一搜索、摘抄或简单算术。
3. 人类完成时间硬下限 3 小时，通常至少 4 小时。
4. 允许 0 至 3 个附件。无附件时说明公开来源或核验范围，有附件时每个附件都要真实可用并进入判断。
5. 已证实事实、合理推断和待确认事项分开，未知价格、权限、数据边界和商业假设不得补造。
6. 允许分阶段交互，本轮可只做摸排、核验或初步比较，后续再形成最终结论。
7. L1 产物格式可以不写，但产物内容必须清楚。指定格式时要有真实使用场景。
8. 不含个人敏感信息、未公开商业秘密或未获授权的数据。

## 当时的第二道语言门禁

1. 使用自然、克制、清楚的中文工作委托，可用“请”“帮我”或直接指令，不统一套开头。
2. 保留请求者当下真正关心的问题，避免规则清单、模型提示词、营销文案和表演式聊天口吻。
3. 允许自然分段，不强制括号、顿号、“等”或题面中的产物格式，不为语言指标增加新内容。
4. 原题存在阶段边界时原样保留，不提前给结论或把本轮改成完整交付。
5. 保留事实、推断和待确认项之间的界线。
6. 返回完整、自洽的题面，不返回局部修改片段。

## 当时的去 AI 事实锁

1. 不增加、删除或改变组织、人物、产品、平台、数字、日期、附件、来源范围、主要任务和产物要求。
2. 保留决策对象、比较范围、关键限制及当前轮次和后续轮次边界。
3. 保留已证实事实、合理推断和待确认事项的区别。
4. 不虚构期限、预算、会议、领导要求、用户情绪、测试结果或已经发生的业务事件。
5. 原题要求来源链接、访问日期、待确认项或补证方法时完整保留。

## 当时的去 AI 表达要求

1. 从真实工作现场自然开口，像同事之间清楚、克制的任务委托。
2. 重新安排句子和信息顺序，不做逐句同义替换，不套“背景—要求—限制—结论”。
3. 主任务保持唯一，多个核验维度可以共同服务同一决策。信息较多时使用自然段和完整句子，不使用项目符号、编号清单或字段名堆叠。
4. 避免机械顺序壳和“全链路、闭环、多维度、赋能”等机器表达，但不因“表格最后”这类自然位置表达退回。
5. 不表演口语，也不虚构“老板催”“挺急”等情节。自然的“麻烦你”“请”“帮我”不作为退回条件。
6. 可以删除重复连接词，但不因压缩遗漏可检查要求，长度与原题大体相当。
7. 标点自然、括号成对，不强行增加括号、顿号或“等”。

## 当前继续保留的较新修正

- 默认国内工作范围、真实附件和事实锚点校验继续保留。
- 发送给豆包前继续做独立可见文本质检，不出现运行状态、接口信息、工具痕迹、乱码或怪异标点。
- “刚传了”“我刚上传了”“这里上传了”“随本消息上传了”等上传元话语删除。
- 不使用“这是……材料”“材料包括……”等独立材料介绍句，材料名称直接进入核验、比较或整理任务。
- 顿号密度只给建议，不作为强制退回条件。
- 使用或省略“你”“我”“我们”均可，不设配额。
- 模型调用失败按重试策略暂停，不以其他模型或原文直发兜底放行。

## 不再作为 L1 语言硬门槛的规则

- 不再禁止“麻烦你”等自然请求语。
- 不再一律禁止“不要自己推测”“不作为最终决策”等自然且必要的证据边界。
- 不再把“在表格最后”误判为机械顺序壳。
- 不再要求每轮只能出现一个主诉求加一个子诉求的表面句法。以一个主要决策是否清楚为准，允许多个从属核验维度共同支撑它。
- 220 至 520 个可见字符仅可作为经验区间，不作为去 AI 改写的结构锁。L1 仍遵守配置中的总体硬边界。
- 允许 1 至 3 个自然段以及段落间空行。
