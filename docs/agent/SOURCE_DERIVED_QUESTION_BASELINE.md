# 题目来源先行与禁止编造基线

> 本文不是三期新增规则。它把二期已经确定的来源约束单独固化，供 L1、L2 和后续档位共同使用。任何新会话开始选题、找附件或生成题面前，都必须先读本文。

## 1. 唯一允许的生成方向

生产顺序固定为：

```text
读取并核验真实材料
  → 从正文抽取对象、事件、时间、指标、记录和规则
  → 建立带来源位置的事实账本
  → 从这些事实中识别一个真实可解决的工作问题
  → 检查附件能否支撑核心任务
  → 生成题面与正式字段
  → 执行两道质检和提交门禁
```

禁止采用相反顺序：

```text
先想行业或故事
  → 编公司计划、岗位压力、预算、时限、内部缺口或试点安排
  → 再找同领域附件拼接
```

附件标题相似、行业相同、关键词相关或发布主体相同，只能说明“可能相关”，不能证明附件支撑题目。材料无法自然形成一个有一定推理难度的工作问题时，当前候选直接作废并重新选材。

## 2. 什么内容可以进入题目

题目中的业务对象、已经发生的事件、数字、日期、状态、限制和主任务依据，只能来自以下位置：

1. 已下载并核验正文的真实附件。
2. 用户明确提供且允许使用的真实输入。
3. 由前两项直接推导出的待核问题，推导过程必须保留证据边界。

请求者可以使用不增加业务事实的中性工作身份，例如“我在做年报数据复核”。角色描述不能带入材料没有提供的公司计划、采购目的、客户范围、内部决策、截止日期或资源条件。

以下内容一律不能为了增强真实感而补写：

- 老板催办、临时会议、上线节点或项目截止时间；
- 预算、订单量、客户数量、内部指标或系统状态；
- 部门争议、岗位权限、供应商关系或采购计划；
- 尚未取得的台账、合同、验收记录、后台数据或现场结果；
- 附件没有披露的结论、原因、责任或生产状态。

材料无法确认的内容只能作为“待确认”“待补材料”或空白回填项。待确认项必须由当前证据缺口自然产生，不能用来给虚构场景收尾。

## 3. 事实抽取最低要求

每份拟用附件必须先记录以下信息：

| 字段 | 要求 |
| --- | --- |
| 文件名称 | 与真实文件一致 |
| 来源 | 原始网页或官方文件地址 |
| 对象 | 文件实际描述的公司、项目、产品、事件或规则 |
| 时间或事件锚点 | 报告期、公告日期、批次、版本或具体事件 |
| 可证事实 | 正文能够直接证明的内容 |
| 来源位置 | PDF页码、表格行号、网页区段或其他稳定定位 |
| 覆盖边界 | 文件无法证明的事项 |
| 文件校验 | 本地路径、文件大小和 SHA-256 |

只有完成正文读取和上述抽取，材料才能进入附件方案。搜索结果摘要、下载页标题、网页片段预览和模型常识不能代替正文抽取。

## 4. 成题判断

一个候选只有同时满足以下条件才能进入题面生成：

1. 核心对象可以在附件正文中定位。
2. 主任务直接处理附件内存在的差异、冲突、口径、变化、规则适用或证据缺口。
3. 至少一份附件提供对象级、事件级或执行级内容。
4. 预期产物能够用附件事实完成主体部分。
5. 缺失信息不会迫使模型补造核心结论。
6. 题目具备真实推理任务，不是简单摘抄、搜索或文件搬运。

政策、法规、指南和行业报告可以承担规则与背景作用。题目要求判断具体设备、客户、门店、合同、项目或生产状态时，还必须具有对应对象级材料。只有公开规则时，任务范围应停在规则整理、材料预审、流程分流或待补清单，不得生成实际对象结论。

## 5. L1 与 L2 的继承关系

来源约束在 L1 和 L2 中完全一致。三期只增加多轮交互，并调整单轮题面的可见密度，不会降低二期的附件真实性和禁止编造要求。

| 项目 | L1 | L2 |
| --- | --- | --- |
| 来源先行 | 必须 | 必须 |
| 对象级证据 | 必须 | 必须 |
| 禁止补造业务事实 | 必须 | 必须 |
| 附件数量 | 1—3份，推荐1—2份核心材料 | 按任务证据链确定 |
| 单轮请求 | 1个主诉求，最多1个直接子诉求 | 按完整流程组织 |
| 题面密度 | 保留当前轮必需的少量事实 | 可承载完整流程所需信息 |

“L1附件采用L2标准”的含义是复用真实性、可读性、可追溯性、对象级证据和事实边界。它不表示把L2的附件数量、数字清单和完整流程压缩进一条L1题面。

## 6. 目录字段

任务类型和一级目录属于飞书选项字段，不能自由生成。

- C列任务类型必须原样选择现有选项，例如 `L1 探索型`。
- D列一级目录必须从飞书当前下拉选项中原样选择。
- 一级目录依据附件正文体现的业务对象和实际任务选择，不能依据编造的使用场景分类。
- 没有匹配选项时放弃选题或重新选材，禁止创造近义类目。
- 提交前必须读取飞书数据校验规则；写入值不在 `conditionValues` 中时停止提交。

二级目录和三级目录虽然可以填写文本，也必须从附件内容和真实任务中概括，不能借此补入附件外场景。

## 7. 典型错误

| 材料 | 错误生成 | 合格方向 |
| --- | --- | --- |
| 两份上市公司年度报告 | 虚构园区更新供应商池，再要求做采购预审 | 比较报告期经营数据、口径变化和年报解释 |
| 召回公告与召回指南 | 虚构门店VIN台账和客户通知项目 | 核对公告初筛条件与VIN最终确认关系 |
| 云平台公开接口文档 | 补写生产租户设备状态和验证结果 | 整理平台规则、字段和待补生产台账 |
| 通用政策或行业指南 | 直接判断具体项目已合规或可上线 | 建立规则核对框架和对象级材料补件清单 |

## 8. 生成、质检与提交门禁

生成提示词必须明确要求“先读材料正文，再出题”。第一道质检至少检查：

- `source-content-not-derived`：题目不能从附件正文抽取；
- `unsupported-fact`：题面出现附件和用户输入没有提供的事实；
- `attachment-not-usable`：附件无法读取、定位或核验；
- `evidence-boundary-missing`：可证事实、推断和待确认项混在一起；
- `category-not-from-options`：一级目录不是飞书合法选项；
- `attachment-role-invalid`：只有规则材料却要求具体对象结论。

命中任一项时停止当前题目的生成链。修复方式只能是删除无来源内容、补充真实对象级材料、收窄任务范围或重新选题。禁止使用模型常识、模板句或相似行业材料自动兜底。

飞书提交前还要执行两层检查：

1. 本地轨迹门禁确认题面、附件、事实账本和正式字段一致。
2. 飞书数据校验门禁确认C、D等选项字段的写入值属于当前下拉选项。

## 9. 新会话启动自检

新会话在生成第一道题前，应能明确回答以下问题：

1. 本次使用了哪些真实附件？
2. 每份附件正文抽取了哪些事实，具体位置在哪里？
3. 主任务由哪项差异、冲突、变化、规则或证据缺口产生？
4. 题面中每个数字、日期、状态和业务背景来自哪里？
5. 哪些内容仍然无法确认？
6. 一级目录对应哪个飞书下拉项，为什么与附件内容匹配？
7. 删除附件后题目是否仍能成立？如果仍然成立，说明题目很可能是先编场景再挂附件，应退回。

任一问题无法回答时不得进入去AI、豆包交互或飞书提交。
