# 垂域高难度题目生产--二期要求

Source: https://xcnhc87xvcni.feishu.cn/wiki/F2ZqwOpzKiST4PkNtDOcHdgSnCc?from=from_copylink

二期训练重点：不再将所有信息体现在题目中喂给大模型，重点在于要求大模型做什么，让大模型自行判断，从而达到训练目的。
- 核心要求：注重真实性，由真实白领场景（工作/生活）出发去设计，逻辑清晰：优先真实表述，从真实场景出发。不要为了堆附件数量而设计过多脱离实际的要求（例：制定旅游计划，不使用app搜索，却去找xx旅游局的后台数据）
- 题目中对于题目的要求可以更详细拆解，尤其是对于更难的任务。附件的描述、角色设定等单纯信息类的内容都需缩短/概括。
本文中的百分比用于同步数据类型优先级，不用于实际交付题目数量的计算。

  - 如:" 公司旗下有一款主打“天然无谷、高肉含量”的中高端猫粮，在天猫和京东的销量还不错。为了寻找新的增长点，决定大力投入B站，试图触达“高知、愿意为科学养宠付费”的年轻养宠人群。但1个月下来，效果惨淡：B站的视频播放量始终徘徊在几百到过千的水平，评论区还充斥着各种不太好的嘲讽；初步排查问题的可能在于，将原本适用于电商平台的“卖点罗列型”文案，直接搬到了以“深度内容”为核心的B站。请帮我深入剖析B站这个平台的内容特点和“社区语言”，设计一个能够引发B站用户自发传播的视频脚本，在设计中，必须体现如何巧妙地绕过用户的“广告防御机制”，将品牌信息自然地融入平台原生内容生态中。"
- 任务类型：
  - 根据任务对大模型的参与度要求作出以下三个层级划分
    - L1探索型、L2流程型、L3系统型
    - 具体查看三、任务类型
- 一级目录:题目所属的一级行业/领域,选自【行业/领域分类】的一级类目（不可修改）
  - 如:互联网与平台业务。
- 二级目录:题目所属二级分类,选自【行业/领域分类】的二级类目（具体类目可根据行业专家的实际经验适当扩充）
  - 如:运营复盘与增长归因。
- 三级目录:题目所属三级分类/具体场景,选自【行业/领域分类】的三级目录（三级目录不受表格上规定的内容限制，可以根据行业专家的实际经验进行调整）
  - 如:多源运营数据复盘。
- 任务概括：简单总结下任务的要点，点明任务的核心即可。
  - 如：进行xx行业分析/xx研究
- 相关附件：附件上传至此字段。是为了完成任务实际需要用到的文件。
  - 逐个上传附件，会更好对照。
- 标注专家工作年限:完成该题目的专家最低从业年限,单位为年;若未工作可写最高学历。
  - 如:3年
- 人类完成时间:预估完成该题目所需人类专家时间,小时为单位(L1题目最低4小时，L2/L3题目最低8小时）
  - 不需要实际完成此任务，只需要预估
- 附件格式:提供原始数据或参考资料的文件格式,如PDF、Word、Excel、TXT、HTML等
- 附件内容:说明附件中包含的主要内容
  - 如:"2024年新能源板块公司财务报表,含利润表与现金流量表;市场调研报告PDF"
  - 需要逐个对附件进行介绍，并指出具体来源（以链接/截图形式），确保真实可信。
- 产物格式:期望输出产物的类型或形式(L1题目可不填）
  - 如:表格、图表、Word、Excel、PPT、商业/管理图表(甘特图、鱼骨图等)
- 产物内容（总结概括）:期望输出产物的大概要求（可包含没有明确设定在题目中的内容）
- 对应【题目】:"某机构需针对 A 股新能源板块公司 2025 年盈利情况开展深度研判，结合板块行业趋势、政策导向及各公司历史经营数据，撰写深度研判报告，现在要求你输出一份不少于20000字word形式的深度调研报告，一份Excel数据可视化图表，明确各公司盈利增长点与潜在风险，且所有预测需基于公开可查数据，不得主观臆断。"
- 【产物内容（总结概括）】:"一份深度调研报告，一份数据可视化图表"
- 做题关键步骤：
  - 作为解题思路和执行过程，要求文字描述清晰、完整、详细并结构化呈现（有序列表），案例如下：
1. 从《AI DC白皮书》中提取超大型/大型/小型AI DC的建设门槛、典型配置（卡数、电力、PUE）、利用率基准（超大型<40%、千卡<60%）、运维复杂度差异，制成对比框架；
2. 从《赋能报告2025》中提炼需求场景分级（预训练需E级万卡、推理/微调用百P级中小中心）及利用率偏低的行业共性原因；
3. 代入"中型金融企业、当前算力使用率4%"的现实约束，结合金融行业算力需求特征（合规推理为主、数据不出域、无需全量预训练）进行适配性判断；
4. 形成明确结论（建议暂不建超大/大型、可考虑小型或租用），列出投入产出核心依据；
5. 识别两项最关键风险（如利用率持续低导致ROI恶化、运维能力不足）并给出缓释建议；
6. 设计分阶段方案：明确各阶段触发条件（如使用率达20%/40%）与验收指标（如卡利用率、推理延迟达标率、TCO对比）。

- 法律与合规：税务律师、跨境合规律师、并购律师、知识产权律师、广告合规专家。
- 医疗与教育：临床专科医生或医学编辑、卫生经济学研究员、留学规划顾问、高校教师或科研项目负责人（要求可提供可核验来源链）。
行业背景最低门槛
必须同时满足：
- 从业年限：相关领域 3 年及以上（如特别硬核的细分领域可 2 年，但需有可证明成果）。
- 基本要求：做过“影响真金白银或重大决策”的工作（投研、尽调、合规评估、方案设计、诊疗/教育决策、增长归因、工程上线等）。
- （后续涉及，具体事宜正在确认）需要提供可核验的产出：至少提供 1 份可脱敏的代表作（报告、方案、备忘录、研究笔记、模型框架、课程、代码项目、前端 demo 或文章）。

L2题目基本规则：
1. 真实性
  1. 需要基于真实的业务场景去设计

    2. 所有题目的附件都必须完全匹配具体的题目需求；
      1. 常见反例：
        1. 不可用国家层级等总体性文件解决个人/公司问题。
        2. 不可有具体的项目但是没有和项目匹配的数据，例如：某个公司要采购某样物资，但是不提供该公司的历史数据，反而提供其他公司或者行业的采购的数据。
2. 题目表述
  1. 题目需采用层层递进的表述形式，如：有什么问题 ➡ 需要干什么 ➡ 需要输出什么 
  2. 禁用没有任何推理的平铺直叙、罗列式写法，如：背景介绍+附件内容/用途+产物的表述形式。此类行文结构无因果、先后、层级逻辑，只是简单列明、信息堆砌，没有一环推一环的推导链条，不具备推理属性。
3. 做题步骤中需体现推理逻辑
  1. 关键做题步骤若有8步，理想情况下，操作类步骤和推理类步骤需要按5:3的比例。
  2. 操作步骤定义：无任何思考性的机械操作，AI只起到归纳/搜集信息/简单计算等作用
    - 仅对附件进行简单梳理或者简单计算
    - 仅做一些信息收集和总结归纳的基础操作
  3. 推理步骤定义：审题分析、逻辑推导、条件关联、思路预判、原理判定、分析决策等步骤
    - 如，用产物1和附件1/搜集的信息结合得到产物2（产物1 + 附件1/搜集的信息→产物2）
    - 如，由题目本身衍生的推理层级：目的1 ➡ 需要达成目的1的必要步骤1 ➡ 达成步骤1的必要步骤2...
    - 保证上下文的连续性：如果是多轮流程型数据，第 N 轮的问题必须深度嵌入前 N-1 轮的上下文信息，考察模型的长文本记忆与状态追踪能力。

五、附件要求
- 数据格式要求以文字类/结构化数据为主，例如：文本类（word/飞书文档/txt/markdown/json），图表类（excel/csv/飞书表格/多维表格），静态网页类，代码/日志/API 文档类，etc。（可以包含少部分音频/视频，用于信息归纳总结，但是不要包含纯音乐类型数据。）
  - 要求带附件题目占所有题目的 70% 以上；同时保留约 30% 无附件或小附件题，用于覆盖敏捷快评中常见的短 prompt、开放检索、无上下文 deep research 任务。
  - 带附件题目占比：pdf 占比要在 25% 以上，excel/csv/xlsx 或其他表格的占比要在 25% 以上，JSON/日志/代码/API 文档占比建议 15% 以上，网页/HTML/截图/产品页面材料占比建议 15% 以上。其它格式都可选取。
  - 附件大小尽量在 10M-500M 之间；L1 探索型可允许小附件或无附件
  - 鼓励文件中包含图片、图表、表格、脚注、扫描件或跨页引用，e.g. 图文交织的 pdf、含多个 sheet 的 xlsx、带公式和条件格式的工作簿、聊天记录 JSON、网页导出 HTML。
六、输出产物格式要求（产物预期不强制体现；增加html产物要求；优先输出可编辑产物格式）
- 题目本身也可以提出复杂要求，60% 以上的题目包含明确对于产物的预期即可。
  - 建议包括的核心产物有：复杂表格，复杂 visualization/demo，复杂商业/经营管理图表（甘特图，鱼骨图，复杂交互式统计图，etc），知识拓扑图，HTML dashboard，前端 demo，游戏 demo，代码项目，工作流配置，trace/runbook，etc。
  - HTML / 交互式网页 / 前端 demo 类产物占比需增加。 一期相关产物要求过少，二期需要提升占比。
    - HTML 题目不能只要求“给出代码片段”，应要求生成可打开、可编辑、可核验的完整产物。
      - 例如：index.html、交互式仪表盘、品牌活动页、游戏 demo、数据分析 dashboard、报告网页、可视化对比页。
    - HTML 产物应明确验收项：是否引用真实数据、是否有交互状态、是否移动端可读、是否有空状态/错误状态、是否没有文字重叠、是否能被用户继续编辑。
  - L2/L3 题目应优先明确要求“可编辑文件底稿 + 决策的摘要”，避免只给截图或不可修改的 PDF。
    - 可编辑文件格式：Word/Docx、Excel/Xlsx/CSV、PPT/PPTX、Markdown、JSON/YAML 等。
  -  L1题目中，产物格式字段可以为空，产物内容字段不能为空，可以保留模糊的产物预期。
    - L2/L3题目，均需体现具体的产物格式、产物内容描述
七、出题要点
1. 题目表达
  1. 真实性：所有题目必须接近真实表达：不要有过多角色设定，直接进入诉求。（用AI合成题目，逻辑漏洞明显，信息密度不真实，整批拒收）
    1. 例如“这是我这个月的报销发票/水单，帮我按时间顺序列一下”“帮我把视频里的英语翻译成男声广东话”“这个 CSV 和上周状态文件帮我整理周报”。（要求专家想象自己真的在用Agent帮助自己完成日常任务，或者在给自己的个人助理派发任务。）
    2. 出题人要在附件内容\产物内容或打分checklist中补齐隐性验收点（不明确列入题目中）
      1. 例如字段含义、排序规则、责任人归属、格式要求、可编辑性、时间锚点等。
    3. 题面中可以保留用户常见写法：不对输出要求做出具体规定，只说“帮我整理/做成可交付版本/能直接发客户”
2. 出题细节
  1. 把任务要求整理成可检查的条件，明确列于题目中
    1. 例如“每个关键数字必须有来源链接”“所有 Excel 公式不得写死为静态值”“HTML 必须可打开并包含交互状态”。
八、题型设计
  大家在出题时要少出这些场景的题目，避免场景过于生活化，难度不足：
    - 买/卖/租：车/房/电脑/手机/表等
    - 买/卖保险
    - 旅游规划
    - 短视频赛道规划
  1. 
    1. 增加真实计算题：ROI、归因解释度、A/B 指标、回测指标、财务敏感性、库存/排班、风险敞口、医疗证据对比等。
      1. 题目中应明确计算口径、分母、分子、聚合顺序、异常值/噪声处理、单位和保留位数。
  2. 允许题目设计“漏答即严重扣分”的要求，训练模型必须完成设计的全部任务。
  3. 事实准确性：
    1. 分清 “可验证的事实” 和 “纯观点”：比如 “地球是圆的” 是可验证事实，“这个电影最好看” 是个人主张，出题时要把两者划清，不能混为一谈。
    2. 会设计有唯一标准答案、卡死条件的题（比如数学公式计算、法规条文）；没有唯一答案、需要权衡的题（比如方案选型、创意设计）；出题要能灵活用这两种约束，覆盖不同考察场景。
    3. 所有信息都能溯源：题目里用到的事实、数据，都要有可查的来源链接，不能凭空造内容。
  4. 专业深度：
    1. 考多步推导和反事实分析：不是考 “背答案”，而是考 “怎么推出来的”，甚至要考 “如果条件变了，结果会怎么样”，倒逼大模型真懂逻辑。
    2. 考多方案对比和权衡（tradeoff）：出那种 “没有完美方案，只能选最合适” 的题，比如 A 方案省钱但慢，B 方案快但贵，让大模型分析利弊、做取舍，考察专业判断力。
    3. 考隐含假设和边界条件：挖那些 “默认成立但没说出口” 的前提，比如 “这个算法在 XX 场景下才有效”，出题要能测出大模型有没有意识到这些边界，会不会踩坑。
    4. 题目具备行业深度：完全符合白领的真实工作场景，题目中一定要包括复杂的信息分析任务（大量数据分析对比），或者重大决策建议，或者对于实效性/事实准确性的测试。
  5. 时效性：
    1. 懂不同知识的更新节奏：比如科技、政策类知识更新快，基础理论类知识更新慢，出题要贴合这个节奏，不能用 10 年前的旧标准考现在的内容。
    2. 给题目加 “时间锚点”：出题时明确标注 “截至 XX 年 XX 月”“按 XX 年最新标准”，让大模型知道该用哪个时间点的知识，避免歧义。（不依赖时效性的题目没必要强加时间锚点）
底线要求：
需求方抽检发现的问题：
1. 附件/数据不真实（不符合现实情况，比如题目设计拿20万投资私募，但实际私募基金有起投门槛100万，不符合现实）
2. AI附件、附件数据和题目要求冲突
3. 附件为宏观信息，query却需要做个人/公司具体决策，导致缺失关键信息无法做出来产物。
4. 此外，多个宏观政策/标准等的附件被用在很多题目中，整体的题目存在雷同的情况。
5. 目前工程类题目偏多，近似蓝领的工作场景，不符合白领的工作场景（不要使用非常复杂的工程题目或者非常难的学术论文出题，可以是简单的设计个app、html之类的）
6. 英文附件过多（后续要附件减少英文附件的比例，后续环节也难以进行）
总体建议：
1. 大家务必先进行内部质检，把控题目场景真实性
2. 附件允许脱敏（偏向为个人敏感信息等），但核心信息和逻辑不允许用AI修改
3. 专家出题环节提供具体到引用附件的下载地址、截图佐证、具体说明，说明尽可能具体
附件来源要求：
1. 附件来源如给出链接，需直接给出具体的附件链接，不可以只是网址首页；如为公司内部文件，需为脱敏后的文件，不要用ai进行合成/后期处理

1. 要做一个公司的具体决策，产品线是否需要往哪个方向走，具体是先要清楚自己公司本身是否具备这样的能力，而不是直接看市场规模。市场很大，但是自身并不具备这样的能力也是白搭。所以首先应该基于公司本身的情况做判断。
2. 第二，题目重点是需要判断哪个具体的场景适合进行发展，那要得到的信息是这些场景的市场规模多大，需要一些什么能力，公司有什么能力是可以迁移的，可以进行适配。但是给到的附件中并没有任何一份跟这些具体场景相关的信息，全部都是一些较为宽泛的行业的信息。并且也没有跟公司相关的信息。

每个关键事实、数字、政策和判断都有可核验出处
链接/文件名/页码/表名/日期齐全，不能只写来源名。
完整性 / coverage
不漏答题面子任务，不遗漏指定产物
输出前有 checklist，逐项对应 prompt。
事实与幻觉控制
不编造数据、论文、公告、文件、API 执行结果
无法确认时标注“未获取到可靠数据”，并说明缺口。
附件引用

能跨附件引用、对比和处理冲突
结论必须回指附件证据，能区分附件证明/推断/待补。
计算与公式
聚合顺序、公式、单位、分母分子正确
输出中间表、公式说明、模型的自检结果。
格式与文件交付
生成真实可编辑文件，而非只描述会生成
文件名、目录、格式、可打开性和内容一致。
行业体例规范
符合短剧、品牌、法律、投研、科研、运营等专业惯例
由对应专家按行业体例打分，泛泛模板不得高分。
长任务执行

能规划、执行、反馈、修正和落盘
有多轮确认、trace、失败处理和最终验收。
HTML/前端可用性

产物不仅美观，而且能运行、能看、能改
页面可打开、交互正常、移动端可读、数据不假。
